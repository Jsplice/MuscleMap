# MuscleMap SVG Asset Briefing

## Goal

Create four premium segmented SVG anatomy assets for a fitness analytics app.

The assets must be suitable for dynamic heatmap coloring based on 0-100 muscle load scores.

## Required assets

1. `male-front.svg`
2. `male-back.svg`
3. `female-front.svg`
4. `female-back.svg`

## Visual style

- Premium fitness analytics style
- Dark-mode optimized
- Clean, modern, app-ready
- Anatomically plausible, not medical-overdetailed
- Strong silhouette
- Clear muscle segmentation
- No organs, blood, or medical textbook appearance
- No excessive veins or fibers
- Visually consistent across all four views

## Technical requirements

- Final delivery as clean SVG files
- Consistent viewBox across all four SVGs
- Each muscle area must be an individual SVG path or group
- Every path/group must have a semantic ID
- Neutral base fills only; app code applies colors dynamically
- No embedded raster images
- No bitmap textures
- No random generated path names
- Must be editable in Figma or Illustrator

## Recommendation

Use AI only for visual direction and mood boards. For production, use a vector illustrator or designer to create clean, segmented, licensed SVG assets.
